# PRD 智能投研助手（ResearchMate）
> **个人投资者的财报解读第一站**——用最轻量的方式，解决个人投资者读不懂财报的核心痛点

## 产品定位与价值主张

### 定位（差异化核心）
- **目标用户**：个人投资者/初级投研人员（区别于机构投资者）
- **功能聚焦**：聚焦"财报解读"单点突破（区别于全链路复杂功能）
- **AI形态**：封装为"财报解析Skill"（区别于通用对话）
- **交互方式**：Web端轻量化 + 未来微信接入（区别于App为主）
- **数据来源**：用户上传PDF + 免费API（区别于自有数据库）

### 核心价值
让个人投资者在 **10分钟内**，通过"一句话查财报"的方式完成对一家公司的基本分析，并以结构化、可视化的方式展示关键指标。

## 三、UI/UX设计规范

### 视觉设计原则
1. **便当网格2.0布局**：仪表盘使用卡片式模块，清晰展示关键指标
2. **深色模式优先**：投研产品用户偏好深色背景，减少视觉疲劳
3. **自然语言输入为主**：搜索框+快捷按钮，降低学习成本
4. **进度可视化**：财报分析过程显示进度条，增强掌控感
5. **结果结构化展示**：关键指标用卡片/表格，而非纯文本

### 合规设计要点
1. **AI生成标识**：所有分析结果标注"AI生成，仅供参考"
2. **风险提示**：明确"不构成投资建议"
3. **不存储敏感数据**：PDF分析后不保存原始文件
4. **简单用户确认**：首次使用展示服务条款

## 六、功能需求（MVP）

### F1 财报智能解读（核心 - 重点优化）

**功能描述**
用户上传财报PDF或通过自然语言查询，系统自动解析并返回结构化数据，展示"财报解读Skill"的标准分析结果。

**迎接条件**
- 用户已登录并同意服务条款
- 上传文件为PDF格式，单个文件 ≤ 50MB
- 支持自然语言查询，如"分析一下这份财报"

**主流程**
1. 用户点击"一句话查财报"或"上传财报"
2. 选择本地PDF文件或输入自然语言查询
3. 系统显示解析进度条（预计完成时间）
4. 解析完成后展示结构化结果页面

**提取字段与展示**
| **字段** | **类型** | **展示形式** | **说明** |
| --- | --- | --- | --- |
| 公司名称 | string | 标题卡片 | 自动识别 |
| 股票代码 | string | 标题卡片 | 自动识别 |
| 报告期 | date | 标题卡片 | 年报/季报 |
| 营收 | number | 指标卡片 | 单位：万元 |
| 净利润 | number | 指标卡片 | 单位：万元 |
| 经营现金流 | number | 指标卡片 | 单位：万元 |
| 负债率 | percent | 指标卡片 | 总负债/总资产 |
| 毛利率 | percent | 指标卡片 | 毛利润/营收 |
| AI分析摘要 | string | 正文卡片 | 200字中文摘要 |
| 情绪评分 | number | 情绪旗帜 | -100到+100 |

**异常处理**
- PDF无文字（扫描件）：提示"该文件为扫描件，OCR功能将在后续版本支持"
- 解析超时60s：提示超时，支持重试
- 未识别到字段：显示"未识别"占位，支持用户手动填写

---

### F2 市场新闻聚合

**功能描述**
接入公开新闻API，按用户关注的公司/行业展示聚合资讯，每日自动小结。

**核心交互**
- 用户可添加/删除关注列表（公司名称或股票代码）
- 首页展示最新N条相关资讯，每条包含：标题、来源、发布时间、情绪分数
- 支持按公司/行业筛选
- 每日 08:00 推送小结通知（可关闭）

**数据来源优先级**（MVP）
1. NewsAPI（免费层）
2. 东方财富数据 API
3. 后续可扩展 RSS 订阅

---

### F3 情绪指标分析

**功能描述**
基于新闻标题对每条资讯做情感分类，并在公司/行业维度聚合为情绪分数。

**分类层级**
- 单条资讯：正面 / 中性 / 负面
- 公司情绪分数：近N天资讯的情感加权平均，展示为 -100 到 +100 的分数

**展示形式**
- 公司详情页展示趋势图（近 30 天）
- 情绪旗帜展示当日情绪：🟢 积极 / ⚪️ 中性 / 🔴 消极

**MVP实现方案**
使用开源轻量模型（如 SnowNLP 或 VADER 中文适配版），无需调用付费LLM接口。

## 九、技术方案建议

| **模块** | **建议方案** | **备注** |
| --- | --- | --- |
| PDF解析 | PyMuPDF + LLM（GPT-4o / Claude） | 提取结构化字段 |
