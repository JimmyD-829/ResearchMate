## 二、背景与目标

### 问题背景

个人投资者在做投资决策时面临三个核心痛点：

- 财报体量大、尓读耐心差，关键指标需要手动翻找
- 财经资讯分散在多个平台，没有统一入口
- 市场情绪判断主观，缺乏数据支撑

### 产品目标

> 让个人投资者在 **10分钟内** 完成对一家公司的基本基本面、新闻动态、市场情绪的全面小结。
>

## 四、用户故事

1. 作为**个人投资者**，我希望上传财报PDF后能快速看到营收、利润等关键指标，**以便判断公司基本面。**
2. 作为**金融分析师**，我希望每天早上打开App就能看到我关注公司的最新资讯摘要，**不用到各大网站一个个翻。**
3. 作为**投资顾问**，我希望快速了解某支股票的当前市场情绪评分，**以便向客户提供更有依据的建议。**

## 五、功能需求（MVP）

### F1 财报智能解读

**功能描述**

用户上传财报PDF，系统自动解析并返回结构化数据。

**迎接条件**

- 用户已登录
- 上传文件为PDF格式，单个文件 ≤ 50MB

**主流程**

1. 用户点击“上传财报”按鈕
2. 选择本地PDF文件
3. 系统显示解析进度（预计完成时间）
4. 解析完成后展示结果页

**提取字段**

| **字段** | **类型** | **说明** |
| --- | --- | --- |
| 公司名称 | string | 自动识别 |
| 股票代码 | string | 自动识别 |
| 报告期 | date | 年报/季报 |
| 营收 | number | 单位：万元 |
| 净利润 | number | 单位：万元 |
| 经营现金流 | number | 单位：万元 |
| 负债率 | percent | 总负债/总资产 |
| 毛利率 | percent | 毛利润/营收 |
| AI摘要 | string | 200字中文摘要 |

**异常处理**

- PDF无文字（扫描件）：提示“该文件为扫描件，OCR功能将在后续版本支持”
- 解析超时60s：提示超时，支持重试
- 未识别到字段：显示“未识别”占位，支持用户手动填写

---

### F2 市场新闻聚合

**功能描述**

接入公开新闻API（如NewsAPI、东方财富等），按用户关注的公司/行业展示聚合资讯，每日自动小结。

**核心交互**

- 用户可添加/删除关注列表（公司名称或股票代码）
- 首页展示最新N条相关资讯，每条包含：标题、来源、发布时间、情绪分数
- 支持按公司/行业筛选
- 每日 08:00 推送小结通知（可关闭）

**数据来源优先级**（MVP）

1. NewsAPI（免费层）
2. 东方财富数据 API
3. 后续可扩展 RSS 订阅

---

### F3 情绪指标分析

**功能描述**

基于新闻标题对每条资讯做情感分类，并在公司/行业维度聚合为情绪分数。

**分类层级**

- 单条资讯：正面 / 中性 / 负面
- 公司情绪分数：迗近N天资讯的情感加权平均，展示为 -100 到 +100 的分数

**展示形式**

- 公司详情页展示趋势图（近 30 天）
- 指数旗帜展示当日情绪：🟢 积极 / ⚪️ 中性 / 🔴 消极

**MVP实现方案**

使用开源轻量模型（如 SnowNLP 或 VADER 中文适配版），无需调用付费LLM接口。

## 六、非功能需求

| **类型** | **要求** |
| --- | --- |
| 性能 | 财报解析 ≤ 30s，新闻加载 ≤ 2s |
| 安全 | 上传财报仅存储在用户自己空间，不训练模型 |
| 兼容性 | Web端首先，支持移动端噪器视图 |
| 可用性 | 每日新闻同步成功率 ≥ 99% |

---

## 七、MVP 边界（不做）

- ✖️ 实时行情数据
- ✖️ 自动交易/下单
- ✖️ 多语言支持（MVP只做中文）
- ✖️ 财报OCR（扫描件）
- ✖️ 社交分享功能

---

## 八、技术方案建议

| **模块** | **建议方案** | **备注** |
| --- | --- | --- |
| PDF解析 | PyMuPDF + LLM（GPT-4o / Claude） | 提取结构化字段 |
| AI摘要 | OpenAI API（gpt-4o-mini） | 控制成本 |
| 新闻聚合 | NewsAPI + 定时任务 | 免费层每日100次请求 |
| 情绪分析 | SnowNLP 或 VADER | 免费、无需GPU |
| 后端 | FastAPI + PostgreSQL | 快速迭代 |
| 前端 | Next.js + TailwindCSS | 快速构建响应式页面 |
| 部署 | Vercel（前端）+ Railway（后端） | 免费起步 |
