## 1. 产品概述

### 1.1 产品定位
ResearchMate 是一款面向个人投资者的智能投研分析平台，通过 AI 技术整合财报解析、行业对标、新闻聚合、情绪分析等多维度数据，为用户提供专业级的投资研究工具。

### 1.2 核心价值主张
- **一站式投研**: 整合多源数据，减少信息搜集时间
- **AI 驱动**: 智能化分析，降低专业门槛
- **数据准确**: 投资决策的基础是准确的数据
- **易于使用**: 简洁直观的界面设计

### 1.3 产品愿景
让每个投资者都能获得机构级别的投研能力。

### 2.1 主要用户群体

## 4. 功能详细说明

### 4.1 财报上传与分析

**功能描述:**
用户上传公司财务报表文件（PDF/Excel/CSV），系统自动提取关键财务指标并进行 AI 分析。

**核心流程:**
```
用户上传文件 → 格式验证 → PDF文本提取(前30页) → AI结构化解析 → 存储结果 → 展示给用户
```

**支持格式:**
- ✅ PDF 文件 (文字版，非扫描件)
- ✅ Excel (.xlsx, .xls)
- ✅ CSV 文件

**提取限制:**
- 最大页数: 30 页 (针对大型年报)
- 最大字符: 15,000 字符
- 文件大小: ≤ 50MB

**输出内容:**
- 公司名称、股票代码
- 报告期间
- 核心财务指标:
  - 营业收入
  - 净利润
  - 经营现金流
  - 资产负债率
  - 毛利率
- AI 生成的摘要分析

**错误处理:**
| 错误类型 | 用户提示 |
|---------|---------|
| 加密 PDF | "该 PDF 文件已加密，无法解析。请提供未加密的版本。" |
| 扫描件 | "该 PDF 为扫描件（图片格式），暂不支持 OCR..." |
| 无法提取 | "无法提取 PDF 文本内容。请确保文件包含可提取的文字。" |
| 格式不支持 | "仅支持 PDF、Excel、CSV 格式文件" |
| 文件过大 | "文件过大（最大支持 50MB）" |

**性能指标:**
- 处理时间: ≤ 30 秒 (288 页平安银行年报实测)
- 成功率: > 90% (文字版 PDF)

### 4.2 行业对标分析

**功能描述:**
输入任意上市公司名称，系统生成该公司的行业对标分析报告，包括财务对比、SWOT 分析、竞争对手比较等。

**核心特性:**
- ✅ 支持任意 A 股/港股/美股上市公司
- ✅ 基于关键词的精确行业分类 (13 大行业)
- ✅ 真实的对标公司名称 (非"行业龙头A")
- ✅ 行业特定的优势/劣势分析
- ✅ 差异化数据生成 (同一公司每次结果一致)

**行业分类规则:**
```

1. 金融服务业: 银行/保险/证券/金融/平安/招商/中信...
2. 新能源与储能: 新能源/储能/光伏/风电/电池/思格/宁德时代...
3. 高端制造业: 汽车/制造/万向/格力/美的/海尔...
4. 科技互联网: 科技/腾讯/阿里/字节/openai/微软/apple...
5. 消费品与零售: 茅台/消费/零售/京东/雅虎/亚马逊...
6. 传统能源: 能源/电力/石油/石化/煤炭...
7. 医疗健康: 医药/生物/健康/恒瑞/药明...
8. 房地产与建筑: 地产/万科/碧桂园/保利...
9. 交通运输: 交通/物流/快递/航空/顺丰...
10. 半导体电子: 半导体/芯片/中芯/台积电...
11. 教育培训: 教育/新东方/好未来...
12. 其他: 现代/综合/新兴 (基于 hash 分配)
```

### 4.3 新闻聚合

**功能描述:**
聚合展示关注公司的最新新闻资讯，支持按公司筛选、情绪标签过滤。

**核心特性:**
- ✅ 支持**所有上市公司** (无预设限制)
- ✅ 基于**行业特征**的新闻生成 (8 大行业模板)
- ✅ **真实媒体来源** (40+ 权威媒体)
- ✅ **定时自动更新** (每日增量)
- ✅ **情绪标签** (positive/negative/neutral)

**新闻生成规则:**

| 行业 | 示例标题模板 | 媒体来源 |
|------|------------|---------|
| 金融 | "数字化转型取得重大进展，线上业务增长+12%" | 21世纪经济报道、财新网... |
| 制造 | "获得比亚迪大额订单，合同金额272亿元" | 界面新闻、每日经济新闻... |
| 科技 | "AI大模型能力再升级，参数量达53,272亿" | 36氪、虎嗅网、钛媒体... |
| 能源 | "新增装机容量达27GW，同比增长+39.8%" | 中国能源报、新能源网... |

**数据真实性声明:**
> ⚠️ 当前版本使用**算法生成的仿真新闻**，具备以下特点：
> - ✅ 高度逼真的业务场景和术语
> - ✅ 合理的数字范围和增长趋势
> - ✅ 真实的媒体名称和时间分布
> - ⚠️ **不适用于正式投资决策或研究报告**
> 
> 未来可选升级：
> - 接入新浪财经 API (低成本)
> - 接入东方财富 API (高质量)
> - 接入 NewsAPI.org (英文为主)

**更新机制:**

| 更新模式 | 触发方式 | 说明 |
|---------|---------|------|
| **每日增量** | 自动/手动 `python run_scheduler.py daily` | 更新旧新闻时间戳 + 补充 5 条新新闻 |
| **强制刷新** | 手动 `python run_scheduler.py force` | 删除旧数据，重新生成 15 条/公司 |
| **按需生成** | 用户查询时自动触发 | 查询某公司时如无数据则自动生成 |

**API 接口:**
```
POST /api/news/update        # 触发每日更新
POST /api/news/force-update   # 强制刷新全部
GET  /api/news?company_name=xxx  # 查询新闻
```

### 4.4 情绪指标分析

**功能描述:**
基于新闻数据和 AI 分析，计算并展示市场情绪指标，辅助判断短期趋势。

**核心指标:**
- **当前情绪分数**: -100 ~ +100
  - > 20: 积极 (positive)
  - < -20: 消极 (negative)
  - 其他: 中性 (neutral)
- **7 日平均**: 近一周情绪均值
- **30 日平均**: 近一月情绪均值
- **趋势图**: 30 天情绪变化曲线

**数据来源:**
- 主要: 新闻文章的情绪分析结果
- 补充: 如无新闻数据，基于公司特征的模拟数据

**计算逻辑:**
```
有真实新闻:
  daily_score = avg(articles.emotion_score)

无新闻数据 (fallback):
  base_score = hash(company_name) % 30 - 15  // -15 ~ +15
  daily_variation = (hash + day_offset * 7) % 25 - 12  // -12 ~ +12
  score = base_score + daily_variation
```

**展示形式:**
- 大号分数显示 (如: 🔵 7.0 分)
- 情绪图标 (🟢积极 / ⚪中性 / 🔴消极)
- 三列指标卡片 (当前 / 7日 / 30日)
- 折线趋势图 (Chart.js)

## 6. 数据准确性要求 (⭐ 最重要)

### 6.1 核心原则
> **投资分析平台最关键的就是数据的准确性。可以没有数据，但必须准确！**

### 6.2 各模块准确性标准

#### 行业对标
- ✅ 行业分类**必须正确** (关键词匹配)
- ✅ 对标公司**必须是真实存在的同行**
- ✅ 财务指标**必须在合理区间**:
  - 营收增长: -10% ~ +80%
  - 利润率: 5% ~ 60%
  - ROE: 5% ~ 35%
  - 负债率: 20% ~ 80%
- ❌ 绝不允许出现:
  - 思格新能源被分为房地产
  - 对标公司叫"行业龙头A"
  - 利润率超过 100%

#### 财报解析
- ✅ 提取的数据**必须来自实际文件**
- ✅ AI 分析**基于实际提取的文本**
- ✅ 明确标注处理范围 ("已提取前 30 页")
- ❌ 不允许编造文件中不存在的数据

#### 新闻数据
- ✅ 标题**必须符合业务场景**
- ✅ 数字**必须在合理范围**
- ✅ 媒体**必须是真实存在**
- ⚠️ 明确标注为"AI 生成"或"系统生成"
- ❌ 不允许出现明显的虚假信息

#### 情绪分析
- ✅ 分数**基于可重复的计算逻辑**
- ✅ 同一公司**每次结果一致** (固定随机种子)
- ✅ 趋势图**反映合理波动**

### 6.3 准确性验证流程
```
代码修改 → 本地测试 (至少 5 家不同行业公司) → 
验证分类正确性 → 验证数值合理性 → 
提交 PR → Code Review → 部署到 Staging → 
回归测试 → 发布 Production
```

| **PE Ratio** | Price-to-Earnings (市盈率) |
